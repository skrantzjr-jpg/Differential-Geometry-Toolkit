  The governing equations of differential geometry can be derived elegantly with the proper intuition. However, for a given n-dimensional manifold, the number of resulting equations scales as 𝑂(𝑛^4). This produces a cumbersome number of equations that is tedious, redundant, and often impractical to solve by hand. This toolkit calculates, optimizes, and stores these expressions.
  Furthermore, especially in applications to general relativity, it is often useful to impose a functional ansantz depending on natural or desirable conditions. For example, it is trivial to show that the ideal ansantz for the Schwarszchild metric takes the form diag(g) = (-f(r), 1/f(r), r**2, (r*sinθ)**2). This toolkit therefore supports functions specified only through their minimal defining properties. It employs a functional dictionary that stores user-defined functions together with their intrinsic coordinate dependence, allowing symbolic derivatives, substitutions, and simplifications to be performed automatically without requiring an explicit functional form. In this way, the toolkit can derive general equations for broad classes of metrics before any particular ansatz is chosen.
  Two extensions of this project naturally follow. Firstly, generating these equations automatically is incredibly useful, but not enough to solve many meaningful problems. It is desirable to analyze these equations and identify couplings, seperabilities, integrability, and classifications of the resulting systems. Furthermore, the program should identify optimal intgration methods and input the systems into a corresponding library which automates this process. Secondly, Riemannian manifolds span a large portion of those useful in general relativity, but not all. Velocity-dependent systems, e.g. trajectories in a magnetic field, require a different framework. Finslerian manifolds have such a utility and the corresponding Finsler Geometry has a much more complex structure than Riemannian. Finslerian manifolds introduce a tangent bundle, which induces at least four known types connection coefficinets and curvature tensors, whereas Riemannian Manifolds only have one. Therefore it is imperative to automate this process in any meaniningful context.
